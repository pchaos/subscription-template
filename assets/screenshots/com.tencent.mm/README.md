## 规则列表

### 规则 1: 视频广告完成后关闭
- **规则 ID**: `1`
- **规则名称**: 视频广告完成后关闭
- **功能描述**: 微信朋友圈视频广告播放完成后自动点击关闭按钮
- **触发条件**:
  - 页面包含 "视频播放器" 描述
  - 出现 "已获得奖励" 文本
  - 出现 "关闭" 按钮
- **动作**: 点击关闭按钮
- **冷却时间**: 3 秒
- **快照 ID**: `1776825889636`

### 规则 2: 视频广告播放中静音
- **规则 ID**: `2`
- **规则名称**: 视频广告播放中静音
- **功能描述**: 微信朋友圈视频广告播放时自动点击音量按钮实现静音
- **触发条件**:
  - 页面包含 "视频播放器" 描述
  - 出现 "X 秒后可获得奖励" 计时器（使用通配符匹配任意秒数）
  - 音量按钮位于 `(815,150)-(875,210)` 位置
- **动作**: 点击音量按钮
- **冷却时间**: 3 秒
- **快照 ID**: 
  - `1776825662026` (未静音状态)
  - `1776830284145` (静音后状态)

## 技术细节

### 音量状态判断
GKD **无法**通过 UI 树判断当前是静音还是有声状态，因为：
- 音量图标位置在静音前后完全相同：`(815,150)-(875,210)`
- 资源 ID、描述文本、可访问性属性都相同
- 只有图标**视觉样式**发生变化（如添加斜线），但这不在 UI 树中体现

**解决方案**: 总是点击音量按钮
- 第一次点击：有声 → 静音
- 第二次点击：静音 → 有声（但受冷却时间保护，不会立即触发）

### 通配符匹配
规则使用 `text*="秒后可获得奖励"` 来匹配任意秒数的计时器：
- ✅ "24 秒后可获得奖励"
- ✅ "19 秒后可获得奖励"
- ✅ "5 秒后可获得奖励"
- ✅ "1 秒后可获得奖励"

### 选择器语法
```typescript
// 完整选择器示例
'[desc="视频播放器"] <<n FrameLayout >n [text*="秒后可获得奖励"] <<n FrameLayout >n ImageView[left=815][top=150][right=875][bottom=210]'

// 说明:
// <<n 表示跨层级查找
// >n 表示直接子节点
// text*="..." 表示包含匹配
// [left=815] 等表示位置属性匹配
```

## 使用方法

1. **本地开发**: 
   - 规则已保存到 `src/apps/com.tencent.mm.ts`
   - 快照已保存到 `assets/screenshots/com.tencent.mm/`
   - 运行 `pnpm build` 构建订阅文件

2. **发布订阅**:
   - 提交代码到 GitHub
   - 通过 GitHub Actions 自动构建
   - 在 GKD App 中添加订阅链接

3. **调试规则**:
   - 在 GKD App 中打开"订阅" → "本地订阅"
   - 启用对应的规则组
   - 打开微信查看效果

## 注意事项

1. **规则优先级**: 关闭规则 (key=1) 优先级高于静音规则 (key=2)
2. **冷却时间**: 两个规则都设置了 3 秒冷却时间，避免重复触发
3. **匹配时间**: 规则在 10 秒内有效，避免过早或过晚触发
4. **重置机制**: `resetMatch: 'app'` 确保每次打开 App 后规则可重新触发
